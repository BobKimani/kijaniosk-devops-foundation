# Reflection

This project demonstrated how to implement safe deployments using a blue/green strategy. I learned how to separate environments, route traffic through nginx, and verify system state using health checks. The most important takeaway was the value of automation, especially how monitoring can detect failures and trigger rollback without manual intervention.

I also understood how to define SLIs and SLOs to make system reliability measurable, and how these metrics connect directly to deployment decisions. Troubleshooting issues with SSH, nginx configuration, and timing during rollback helped me better understand real-world deployment challenges.

Overall, this project strengthened my understanding of deployment safety, observability, and operational reliability in a production-like environment.
